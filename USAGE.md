# Twitter Like Catcher V2.0 - 使用方法指南

## 概述

Twitter Like Catcher是一款Chrome扩展程序，能够自动捕获您在Twitter上点赞的推文，并将它们同步到云端数据库。本指南将帮助您快速上手并充分利用这个工具。

## 安装与配置

### 1. 开发环境搭建（开发者）

如果您是开发者，需要从源代码构建扩展：

```bash
# 克隆项目
git clone <repository-url>
cd twitter-like-catcher-(cloud-sync)

# 安装依赖
npm install

# 如果遇到依赖冲突错误，可以尝试：
# npm install --legacy-peer-deps
# 或者使用 yarn：
# npm install -g yarn && yarn install

# 配置环境变量
cp .env.example .env.local
```

编辑 `.env.local` 文件，添加您的Supabase配置：

```env
SUPABASE_URL=your_supabase_project_url
SUPABASE_ANON_KEY=your_supabase_anon_key
```

### 2. 构建与加载扩展

```bash
# 开发模式（热重载）
npm run dev

# 生产构建
npm run build
```

**在Chrome中加载扩展：**
1. 打开 Chrome 浏览器，访问 `chrome://extensions/`
2. 开启右上角的"开发者模式"
3. 点击"加载已解压的扩展程序"
4. 选择项目目录（构建后的dist文件夹）

### 3. 直接安装（最终用户）

如果您获得了已构建的扩展包：
1. 解压扩展文件到本地目录
2. 按照上述步骤2-4加载扩展

## 日常使用流程

### 1. 基本使用

1. **安装扩展** - 将扩展成功加载到Chrome浏览器
2. **访问Twitter** - 打开 [twitter.com](https://twitter.com) 或 [x.com](https://x.com)
3. **点赞推文** - 正常点赞您感兴趣的推文
4. **查看收集** - 点击浏览器右上角的扩展图标查看已捕获的推文

### 2. 功能特性说明

#### 自动捕获
- ✅ **即时捕获** - 点赞后立即提取推文信息
- ✅ **智能提取** - 自动获取推文ID、作者、内容、URL等信息
- ✅ **引用推文** - 支持捕获包含引用推文的复杂内容

#### 智能标签
- 🔍 **自动分类** - 系统根据推文内容自动添加相关标签
- 🏷️ **标签类型** - 包括react、typescript、ai、webdev、design、database、opensource等
- 📊 **快速筛选** - 通过标签快速找到特定主题的推文

#### 云端同步
- ☁️ **实时同步** - 数据立即保存到Supabase云端数据库
- 📱 **跨设备访问** - 在任何设备上登录都能查看您的推文收藏
- 💾 **数据安全** - 云端存储，永不丢失

### 3. 用户界面操作

#### 弹窗界面功能

**顶部区域：**
- **标题** - 显示"Twitter Like Catcher"
- **刷新按钮** - 点击手动刷新推文列表
- **状态指示** - 加载中显示旋转图标

**主要内容区域：**
- **推文列表** - 按时间倒序显示最近捕获的推文
- **推文卡片** - 显示推文作者、内容、标签等信息
- **空状态** - 当没有推文时显示提示信息

**底部区域：**
- **版本信息** - 显示当前版本号

#### 操作说明

**刷新数据：**
- 点击右上角的刷新图标
- 系统会重新从云端获取最新数据
- 适用于多设备同步后更新本地显示

**查看推文：**
- 点击推文卡片可查看完整内容
- 推文按捕获时间倒序排列
- 支持滚动查看更多推文

## 数据管理

### 1. 数据查看

**在扩展中查看：**
- 打开扩展弹窗即可查看所有已捕获的推文
- 支持查看推文的标签分类
- 显示推文的捕获时间

**通过Supabase Dashboard：**
1. 访问您的Supabase项目仪表板
2. 进入Table Editor查看tweets表
3. 可以查看、编辑、删除推文数据

### 2. 数据导出

**CSV导出：**
1. 在Supabase Dashboard中进入tweets表
2. 点击"Export"按钮
3. 选择CSV格式导出
4. 下载包含所有推文数据的文件

**数据字段包括：**
- id: 推文唯一标识符
- author_name: 作者姓名
- author_handle: 作者账号
- tweet_text: 推文内容
- tweet_url: 推文链接
- tags: 自动标签数组
- quoted_tweet: 引用推文信息（如果有）
- captured_at: 捕获时间

### 3. 数据同步状态

**扩展图标状态：**
- 🟢 **绿色背景 + ✓** - 成功捕获并同步
- 🔴 **红色背景 + ERR** - 同步失败
- ⚪ **无文字** - 正常待机状态

**网络连接：**
- 确保网络连接正常
- Supabase服务可用
- 如果同步失败，检查网络和API配置

## 故障排除

### 1. 常见问题

#### 扩展无法捕获推文
**可能原因：**
- Twitter页面结构变化导致选择器失效
- 网络连接问题
- Supabase配置错误

**解决方案：**
- 检查浏览器控制台是否有错误信息
- 确认Supabase URL和Key配置正确
- 尝试重新加载扩展

#### 数据不同步
**可能原因：**
- 网络连接中断
- Supabase服务异常
- 权限配置问题

**解决方案：**
- 检查网络连接
- 确认Supabase项目状态正常
- 检查数据库权限设置

#### 标签不准确
**可能原因：**
- 关键词匹配算法限制
- 推文内容特殊
- 标签库需要更新

**解决方案：**
- 检查keywordService.ts中的关键词配置
- 根据需要自定义关键词
- 考虑添加更多关键词变体

### 2. 调试信息

**查看日志：**
1. 右键点击扩展图标
2. 选择"管理扩展程序"
3. 点击"背景页"查看后台日志
4. 在Twitter页面按F12查看内容脚本日志

**重要日志信息：**
- "Twitter Like Catcher content script loaded." - 内容脚本加载成功
- "Sending liked tweet to background script" - 成功捕获推文
- "Tweet saved successfully" - 数据保存成功
- 错误信息会显示具体错误详情
